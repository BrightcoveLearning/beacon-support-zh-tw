介紹

為了提供客戶用戶管理系統 (UMS) 和 Beacon 之間的集成，Brightcove 實施了 Open ID Connect (OIDC)，以提供最可靠的身份驗證方式。

Open ID Connect (OIDC) 是 OAuth 2.0 協議之上的一層，用於對 Auth0 等服務進行外部身份驗證，它允許客戶端根據授權服務器執行的身份驗證來驗證最終用戶的身份。

以下是我們OIDC集成的突出特點：

  • 現有的第三方數據庫可用於驗證Beacon用戶。
    •   
  • 信標用戶通過其信標應用程序註冊，認證和更改密碼。
  • 信標應用程序支持應用程序購買。
    •  
  • 現有應用程序和Beacon應用程序共享相同的身份驗證數據庫。
  • 新觀眾可以在第三方系統中註冊。
  • 設備配對工作流程（第二個設備，在屏幕上輸入代碼以驗證電視應用程序）。
  • OIDC 提供商支持的社交登錄。
  • 支持信標流並發限制。
  • OIDC 提供商可能會提供除用戶名和密碼之外的其他身份驗證方法（多因素、無密碼、設備配對、谷歌帳戶配對）。

局限性

實施OIDC Auth0提供程序

以下是Brightcove Beacon中實現OIDC Auth0提供程序要遵循的詳細步驟：

  1. 輸入您的Brightcove Beacon帳戶。
  2. 點擊右上角的齒輪按鈕。
  3. 點擊身份驗證提供者左側菜單上的按鈕。
  4. 添加驗證端點： / account / social_login。
  5. 為了驗證類型字段中，選擇openid_connect。
  6. 填寫每個OpenID Connect設置的字段：
    填寫此表格字段的大部分信息都在您的Auth0 帳號 .
    • 一種）基本網址：請與您的Brightcove代表一起查看此字段的值。
    • B）撤消刷新網址：設置oauth/撤銷
    • C）配對代碼網址：設置oauth/設備/代碼
    • D）有聽眾：請與您的Brightcove代表一起查看此字段的值。
    • E）用戶信息網址：設置用戶信息
    • F）訪問令牌網址：設置oauth/令牌
    • G）登出網址：設置v2/註銷
    • H） Jwks地點：是個JSON Web密鑰集位於高級設置>端點您的應用程序中的標籤Auth0帳戶。
    • 一世）刷新令牌網址：設置oauth/令牌
    • J）授權網址：設置授權
    • K）範圍：請與您的Brightcove代表一起查看此字段。
  7. 點擊保存設置按鈕。
