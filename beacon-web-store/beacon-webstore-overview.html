介紹

Beacon 網上商店是一種配置、功能和 API 集成，使 Beacon 被許可人能夠在自帶 (BYO) 網上商店中實施訂閱。內容所有者在網上商店中實現用於租賃和銷售的店面和業務邏輯以及支付系統集成。Beacon API為商店提供了在Beacon CMS中添加，刪除和編輯所產生的權利更改的接口。Beacon 應用中定義訪問和顯示內容標題的邏輯沒有改變。

信標中未管理或跟踪所有財務報告，儀表板和納稅的實施，這是內容所有者的責任。

燈塔商店組件

所有Beacon Store購買都是通過網上商店進行的。銷售模式要求查看者訪問客戶網上商店，購買內容，然後作為單獨的操作在其支持的設備上打開 Beacon 應用程序。

所需的組件是：

信標存儲工作流程和配置

在引入 Beacon 之前，您很可能已經擁有一個商業網站作為其業務的一個組成部分，並希望將 Beacon 訂閱添加到您的網上商店產品中。網絡商店由三個部分組成：

  1. 網上商店 - 網上商店是您組織的商業站點。它介紹了產品，定價和購買模型。
  2. 付款處理器 -付款處理器是一種處理交易，從信用卡，借記卡和電子錢包接受付款，將付款存入組織的銀行帳戶並將收據發送給購買者的系統。
  3. 用戶管理系統 - 您的組織在其 UMS 中維護其註冊讀者的列表。

Beacon Store 集成有兩個開發活動

  1. 信標從網絡商店接收財務交易通知 -從網上商店銷售和取消Beacon訂閱均報告給Beacon API。信標根據這些通知添加或刪除權利。
  2. 連接並同步Beacon和Web Store用戶管理系統 -Webstore UMS用戶記錄包含主記錄和用戶，並且是存儲的用戶數據的源。在信標UMS中為每個註冊的顧客創建代理用戶。信標一接收到交易通知，便將權利分配給代理用戶記錄。存儲UMS中的更改（例如添加和刪除用戶）將應用於信標UMS中的匹配用戶記錄。

信標存儲配置

Beacon商店是與Beacon集成的，現有的，客戶擁有的和運營的BYO網絡商店，其目的是將Beacon訂閱添加到該網絡商店中出售的產品列表中。藝術組織提供了一個示例來說明支持該目標的兩種主要銷售模式。

藝術組織目前通過其網上商店出售與表演相關的訂閱，門票和實物商品。因此，存在現有的網絡商店、用戶管理、支付處理和權利跟踪系統來管理該交易。

一些組織已決定通過包括對這些相同事件和錄製內容的虛擬訪問來擴大他們的受眾。他們還希望在他們的網絡商店中整合整個商務管理流程。

一部分組織希望通過其網絡商店獲得相同的結果，並希望出售與應用商店中的應用購買相同的產品（Apple Store，Google Play，Amazon，Roku和與Beacon Web應用集成的Stripe） 。

僅限網上商店購買

僅 Web 商店的工作流實現如圖 1 所示。客戶帶來自己的網上商店、支付處理器和 UMS。集成點是：

  • UMS 代理用戶 - 註冊用戶的完整記錄保存在客戶的BYO UMS中，這是用戶賬戶和用戶賬戶管理的真實來源。登錄時的用戶身份驗證發生在 BYO UMS 中。身份驗證或訂閱後，將在信標UMS中創建代理用戶。當用戶從 UMS 中刪除時，代理用戶也從 Beacon 中刪除。

    身份驗證和帳戶管理的同步使用 OIDC ( 打開 ID 連接 ) 協議，OAuth 2.0 的擴展。您必須提供具有本地 OIDC 支持的系統或更新您的 UMS 接口以支持 OIDC。

  • 購買通知 -您修改了網上商店，將Commerce Events的通知發送到Beacon Entitlements API。該通知包括用戶 ID 和產品 SKU。當 Beacon 收到通知時，用戶記錄中權利的適當更新會記錄在 Beacon UMS 中。因此授予或撤銷使用戶能夠在 Beacon 應用程序中查看內容的權利。技術權利 API 接口定義在信標集成詳細信息

配置詳細信息：

  • webstore 和 Beacon 中的訂閱定義必須使用相同的 SKU 名稱進行配置。
  • 網絡商店通知被發送到從 Beacon CMS 顯示的網絡鉤子 URL。

應用程序購買工作流中的 Web 商店的實現如圖 2 所示。在這種模式下，可以從Amazon，Apple，Google和Roku商店以及您的網上商店進行應用內購買。您負責：

用戶管理系統

Beacon 商店的典型用例是將 Beacon 與預先存在的、已建立的網絡商店進行集成。較不常見的是通過新的網絡商店啟動新服務以管理商務。

外部連接的UMS

既有的網上商店始終與用戶管理和付款處理系統集成在一起。在這種情況下，您的目標是將Beacon集成到預先存在的商業配置中。您的 UMS 將繼續作為身份驗證和用戶記錄管理點。因此，集成將使用客戶 UMS 來驗證和授權 Beacon 用戶。用戶的電子郵件地址用作帳戶 ID，並且權利與該帳戶相關聯。所有用戶個人信息數據都存儲在BYO UMS中。

外部BYO UMS系統使用OIDC接口協議與Beacon進行接口。Open ID Connect是OAuth的擴展，用於標準化身份令牌架構。

移動和Web UMS配置

  1. 用戶選擇登錄 Beacon App
  2. 應用程序與OIDC提供程序進行交互，以引導用戶完成登錄流程。成功後，將向應用程序提供訪問，刷新和身份令牌
  3. 應用程序使用訪問令牌連接到Beacon後端
  4. 信標後端驗證訪問令牌

連接和SmartTV UMS配置

  1. 用戶選擇登錄 Beacon App
  2. 應用要求輸入設備代碼
  3. 用戶轉到移動設備或 Web 設備，輸入身份驗證 URL 和設備代碼
  4. 設備與OIDC提供程序進行交互以對用戶進行身份驗證
  5. 聯網或智能電視連續輪詢OIDC提供商，以查明是否向用戶提供了通過身份驗證的身份（通過身份驗證時），訪問，刷新和身份令牌
  6. 應用程序使用訪問令牌連接到Beacon後端
  7. 信標後端驗證訪問令牌

信標用戶管理自動同步

將現有 UMS 與 Beacon 集成時，必須在 Beacon UMS 中創建代理用戶，以便可以將 Beacon Store 銷售通知創建的權限分配給適當的用戶。

作為客戶 UMS 中事件的結果，用戶在 Beacon 中得到維護。通過 OIDC 接口，當用戶在外部 UMS 中進行身份驗證或將授權發送到 Beacon 時，將在 Beacon 中創建用戶。無需手動同步。

信標商店獲利模型

信標存儲可配置為支持SVOD事務。配置詳細信息：

點播配置

信標配置

  1. 在信標中創建SVOD計劃
  2. 在 Beacon 中將狀態設置為 UNPUBLISHED
  3. 在信標中為計劃定義SKU
  4. 將內容分配給計劃

網店配置

  1. 在網上商店和網上商店付款系統中創建SVOD計劃/產品
  2. 演戲與信標中使用的名稱相同
  3. 將Beacon中的SKU複製到網上商店付款系統中的SKU ID中
