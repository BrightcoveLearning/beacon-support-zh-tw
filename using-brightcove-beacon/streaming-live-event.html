概述

要將 Live 事件流式傳輸到您的應用程序，您必須按照一系列步驟連接 Beacon Studio Live 帳戶和 Beacon Classic。

該過程的高級視圖是：

  1. 在 Beacon Classic 中，創建一個事件和一個播放列表。
  2. 在 Beacon Studio 中，訪問 Live 模塊以創建您的事件並獲取作業 ID。
  3. 在 Beacon Classic 中，使用來自 Beacon Studio 的作業 ID 註冊您的實時事件。
  4. 直播視頻將開始在 Beacon 應用程序中流式傳輸。

在本文檔的最後一部分，您將看到通過使用靜態入口點在Live中創建不同類型的事件的步驟。這允許可以激活和停用並且只需要配置一次的重複性實時事件。

本文檔的其餘部分將提供上述每個步驟的詳細信息。

在 Beacon Classic 中創建現場活動

按照以下步驟在 Beacon Classic 中創建事件。

僅激活帳戶信標經典現場將有播放列表和活動在選項卡部分可用。如果您沒有激活它們，請聯繫 Brightcove 支持。
  1. 在 Beacon Classic 中，單擊活動選項卡以開始設置事件。
  2. 點擊新增活動。
  3. 給直播活動一個名稱和活動開始日期。
  4. 在裡面文字數據標籤，填寫您的活動信息。
  5. 點擊非文字數據標籤。
  6. 提供圖像人像海報和風景海報符合以下要求。你不需要提供圖像縮略圖和寬幅橫幅 .
    圖像 要求
    人像海報 海報圖片（2：3）
    最低限度 寬度：1000像素 高度：563px
    最大 寬度：4000像素 高度：2250像素
    風景海報 風景圖片（16：9）
    最低限度 寬度：1000像素 高度：563px
    最大 寬度：4000像素 高度：2250像素
  7. 點擊權利與日程安排標籤，如下所示：
  8. 為以下字段添加值：
    • 權限類型：選擇之間的權利類型 廣告， 訂閱 或者 自由.
    • 開始時間：設置活動的開始日期。
    • 時間結束：設置活動的結束日期。
    • 最大流：設置允許查看實時事件的最大流數。默認數字為 0（零）。
    • 要發布的設備：添加您要發佈到的設備。點擊選擇/取消選擇所有設備查看可用設備的列表。
    • 允許的位置：如果沒有限制，請選擇要流式傳輸內容的位置 世界.
    • 遭到拒絕的地點：選擇您不想流式傳輸內容的位置。
  9. 點擊建立新活動按鈕創建您的活動。

將 Beacon Classic 事件添加到播放列表

創建事件後，下一個任務是將事件添加到播放列表中。以下步驟將引導您完成該過程。

  1. 在 Beacon Classic 中，單擊播放列表標籤。
  2. 點擊新增播放清單按鈕。
    您需要在 Beacon Classic 中創建一個包含您的現場活動的播放列表。不要使用Video Cloud中的播放列表。使用 Video Cloud 播放列表可能會覆蓋和刪除您的 Beacon Classic Live 錄音。
  3. 在播放清單，從內容類型標籤中，選擇大事記標籤。
  4. 在內容列表中查找您的事件，然後單擊（+）按鈕將其添加到左側的播放列表中。您可以按照您希望它們在應用程序中的顯示方式來定位事件的順序。
  5. 在播放列表中調整播放列表的設置（時間，設備，位置等）。權利與日程安排標籤，就像您處理活動一樣。
  6. 點擊將更改保存在播放列表中按鈕。
  7. 點擊工具屏幕右上方的（扳手）按鈕。然後清除緩存。

在 Beacon Studio 中創建現場活動

按照以下步驟創建將流式傳輸到 Brightcove Beacon 應用程序的實際直播活動。

有關所有 Live 模塊組件的更多信息，請訪問管理實時事件文檔。
  1. 登錄 Beacon Studio 並點擊居住在主導航中。
  2. 來自管理活動頁面上，單擊建立活動。這創建實時事件頁面將打開。
  3. 輸入事件詳細信息。
    • 活動名稱 -活動名稱；如果現場直播事件另存為視頻資產，則此名稱也將出現在“媒體"模塊中（最多255個字符）
    • 簡短的介紹 - 事件描述（最多 250 個字符）
    • 標籤 - 與事件關聯的標籤；如果將直播活動另存為視頻資產，則會為視頻分配標籤（最多 128 個字符和 1200 個標籤）
    • 參考編號 -事件的參考ID；必須是唯一的（最多150個字符）
  4. 點擊允許的如果您希望將直播活動流式傳輸到社交媒體平台，請單擊不允許。這可以做到一旦事件開始流式傳輸或通過使用社交模塊 .
  5. 選擇一個實時攝取配置文件 .下拉列表將列出所有實時攝取配置文件。攝取配置文件將控制創建的實時再現的質量和數量。點擊管理個人資料鏈接將在另一個瀏覽器標籤中打開提取配置文件編輯器。請注意，您必須是帳戶管理員才能訪問提取配置文件編輯器。點擊演繹鏈接以查看將使用選定的攝取配置文件創建的演繹形式。
    三層的碼率和計費明細如下：
    • 第 1 層允許高達 5,500 Kbps 並按 1.0x 小時收費
    • 第 2 層允許高達 10,000 Kbps 並按 1.5 倍小時收費
    • 第 3 層允許高達 20,000 Kbps 並按 2.0x 小時收費
  6. 選擇完成後將事件轉換為視頻資產將現場活動的 VOD 版本保存到您的 Beacon Studio 帳戶。選擇此選項後，所選內容的最高質量再現實時攝取配置文件用作 VOD 視頻的源。創建再現時使用帳戶的默認攝取配置文件。請注意，直播活動結束後，VOD版本將不可用，一旦提取過程完成，視頻將可用。
    筆記：使用重複事件時不支持此選項。
  7. 點擊高級選項並查看高級選項。僅其中之一將需要您的注意。
  8. 單擊啟用流狀態通知複選框並添加來自 Beacon Classic CMS 的通知 URL（您的 Brightcove 支持代表將提供此 URL）。
    這通知網址設置偵聽器以觸發事件中的進一步操作。有關 VOD 和 Live 事件的狀態通知和狀態更改的更多信息，請訪問實時 API：通知文檔 .
  9. 點擊建立活動開始直播活動。控制室頁面將打開，其中包含有關現場活動的詳細信息，包括流媒體端點（RTMP URL）和流名稱您的編碼器將使用它。處理您的流大約需要一分鐘。
    要在直播平台上開始流式傳輸，您需要一個編碼器，有關如何使用編碼器的信息，請閱讀以下文檔：

    一步步：將直播模塊與 Telestream Wirecast 結合使用

    一步步：將直播模塊與 Open Broadcaster Software (OBS) 結合使用

  10. 在裡面控制室，複製作業編號和視頻標識來自測試活動面板，您將需要兩者在 Beacon Classic 平台設置中配置 Live 事件。
  11. 接下來，返回 Beacon Classic 並單擊事件標籤。選擇您之前在那裡創建的事件，然後在基本數據設置粘貼 作業編號 您的現場活動。
  12. 轉到溪流標籤，然後通過點擊+添加新流按鈕。
  13. 在裡面流視頻 ID字段，粘貼視頻標識來自您之前創建的 Beacon Studio 活動。還填寫有關流的以下信息：
    • 選擇可以發布視頻的設備數量。
    • 設置流類型到居住從下拉菜單中。
    • 然後點擊更新事件按鈕，然後再離開頁面。
  14. 點擊工具屏幕右上方的（扳手）按鈕。然後清除緩存。
  15. 打開您的 Beacon 應用程序並選擇您的活動（圖像上的活動名稱和圖片僅供參考）。
  16. 點擊玩按鈕，您現在應該可以在 Beacon Classic 應用程序中看到來自 Beacon Studio 的實時事件流。

活動結束後，您可以將其全部或部分保存為剪輯，該剪輯將成為 Beacon Studio 中的視頻資產或可在 Beacon Classic 中顯示的可播放流。本文檔的下一部分詳細介紹了該過程。

通過現場活動創建片段

如果您想保留視頻以供稍後作為 VOD 錄製的視頻流式傳輸，您必須創建 Beacon Classic Live 事件的剪輯。

Live 模塊提供從已完成的事件創建視頻剪輯的能力。現場活動進行過程中也可以創建剪輯。

有關完整的裁剪功能列表，請查看使用實時模塊創建視頻剪輯文檔。

  1. 在實時模塊中，轉到創建剪輯左側面板中的標籤。
  2. 使用剪輯按鈕或快捷鍵，設置剪輯的起點和終點。您也可以直接點擊節省按鈕以保留整個視頻。
  3. 輸入剪輯視頻屬性。
    • 剪輯名稱 -剪輯名稱
    • 簡短的介紹 - 剪輯說明
    • 標籤 - 與剪輯相關聯的標籤
    • 參考編號 -分配給剪輯的參考ID
    • 刪除源廣告插播內容 - 此選項刪除剪輯內觸發的廣告中斷，僅保留節目內容
      筆記：刪除源廣告中斷複選框僅在流啟用 SSAI 時出現。
  4. 點擊創建剪輯。剪輯的轉碼過程將開始。使用該帳戶的默認提取配置文件對片段進行轉碼。

將您的剪輯作為可播放的流視頻添加到 Beacon Classic

在這裡，您將使用新創建的剪輯視頻的 ID，並使其成為可在 Beacon Classic 中播放的流。

  1. 在 Beacon Studio 的 Live 模塊中，複製號碼ID您創建的剪輯。
  2. 在 Beacon Classic 平台中，轉到活動選項卡並選擇流菜單。您將看到以下表格：
  3. 粘貼ID您剛剛從Studio的Live平台複製到了流視頻ID場地。
  4. 來自流類型下拉式變更居住至溪流。
  5. 點擊基本數據標籤並更改地位從已結束至完成的。
  6. 離開之前，請點擊更新事件頁面底部的按鈕。這將使玩 Beacon 應用程序中可用的按鈕以查看事件的記錄。

（靜態入口點）SEP Live事件

SEP Live Events 是重複發生的事件，您可以在您的事件一遍又一遍地開始時激活/停用，例如對於每週節目。此事件可作為所有人的一個實時作業 ID可以傳輸到 Beacon App 中的多個頻道。

  • 請注意，創建重複事件時，會在“媒體"模塊中自動為其創建視頻。做不是刪除此視頻，因為每次事件再次發生時都需要它。
  • 應在編碼器啟動之前激活重複事件。

按照以下步驟在 Beacon Classic 中創建 SEP（靜態入口點）直播事件。

  1. 在實時平台中創建一個新活動。有關如何創建事件的更多信息，請閱讀創建一個新事件的部分 使用實時模塊創建和管理實時事件 文檔。
  2. 展開高級選項，然後啟用使用靜態入口點（SEP）將其創建為重複事件選項。有關 SEP 的更多信息，請參閱處理重複事件的部分 使用實時模塊創建和管理實時事件 文檔 。
  3. 如果您已經有SEP事件，請點擊支持標籤，您將能夠激活/停用它來自實時控制室。
  4. 複製您的現場工作ID。
  5. 返回 Beacon Classic 平台並通過粘貼作業編號重複發生的事件。還提供一個名稱和活動開始日期。
  6. 活動的日期和時間必須採用以下格式。
  7. 點擊更新事件按鈕。
Beacon Classic CMS 正在自動化地位 Live 事件的轉換，但在靜態入口點中，您可能有多個 Beacon Classic 事件與同一個 SEP 相關聯。如果自動化出現問題，因為自動化運行較晚且重疊，則可以通過手動更改地位。例如，從可用的到完成的在裡面地位場地。

您可以在 Beacon Classic 平台中為您想要的所有直播活動執行此過程。所有這些都將通過同一Live傳輸進行流式傳輸，但是每個事件都具有在事件設置中調整的不同日期和時間。
