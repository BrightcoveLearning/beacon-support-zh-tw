    串流即時事件

    在本主題中，您將學習如何將直播活動串流至布萊特灣信標應用程式。

    概覽

    若要將即時活動串流至您的應用程式，您必須遵循一系列步驟來連接您的視訊雲端 Live 帳戶和布萊特灣信標。

    該過程的高級視圖是：

    1. 在布萊特灣信標中，創建一個事件和一個播放列表。
    2. 在視訊雲端中，存取即時模組以建立您的事件並取得工作 ID。
    3. 在布萊特灣信標中，使用視訊雲端的工作 ID 註冊您的即時活動。
    4. 直播影片將在布萊特灣信標應用程式中開始串流。

    在本文件的最後一節中，您會看到使用靜態進入點建立不同類型的事件在 Live 中的步驟。這允許可以啟用和停用的週期性即時事件，而且只需要設定一次。

    本文件的其餘部分將提供上述每個步驟的詳細資料。

    在布萊特灣信標建立直播活動

    按照以下步驟在布萊特灣信標中創建一個活動。

    只有在 B rightcove Beacon 直播啟動的帳戶才能在標籤頁中顯示播放清單和活動。如果您沒有啟動，請聯繫布萊特灣支援部門。
    1. 在Brightcove Beacon中，單擊大事記標籤以啟動事件的設置。
    2. 點擊新增活動。
    3. 給直播活動一個名稱和活動開始日期。
    4. 在裡面文字數據標籤，填寫您的活動信息。
    5. 單擊非文本數據選項卡。
    6. 提供圖片人像海報和風景海報符合以下要求。您不需要為「縮圖」和「寬廣」橫幅提供影像。
      影像 需求
      人像海報 海報影像 (2:3)
      最小值 寬度：1000 像素 高度：563 像素
      最大值 寬度：4000 像素 高度：2250 像素
      風景海報 橫向影像 (16:9)
      最小值 寬度：1000 像素 高度：563 像素
      最大值 寬度：4000 像素 高度：2250 像素
    7. 點擊權利和計劃標籤，如下所示：
    8. 新增下列欄位的值：
      • 權利類型：選擇權利類型廣告，訂閱要么自由。
      • 開始時間：設定活動的開始日期。
      • 時間結束：設定活動的結束日期。
      • 最大流：設定檢視即時事件允許的串流數目上限。預設數字為 0 (零)。
      • 要發布的設備：新增您要發佈的裝置。請點擊選擇/取消選擇所有設備查看可用設備列表。
      • 允許的位置：如果沒有限制進入 World ，請選取您想要串流內容的位置。
      • 拒絕的地點：選取您不想串流內容的位置。
    9. 點擊建立新活動按鈕創建您的活動。

    將布萊特灣信標事件新增至播放清單

    建立事件後，接下來的工作就是將事件新增至播放清單。下列步驟會引導您完成該程序。

    1. 在Brightcove Beacon中，單擊播放清單標籤。
    2. 點擊新增播放清單按鈕。
      您需要在布萊特灣信標中建立一個包含您的直播活動的播放清單。不要使用Video Cloud中的播放列表。使用視訊雲端播放清單可能會覆寫並刪除您的布萊特灣信標即時錄製內容。
    3. 在播放清單，從內容類型標籤中，選擇大事記標籤。
    4. 在內容清單中尋找您的事件，然後按一下 (+) 按鈕，將其新增至左側播放清單。您可以按照您希望在應用程式中顯示事件的方式來定位事件的順序。
    5. 在播放列表中調整播放列表的設置（時間，設備，位置等）。權利和計劃標籤，就像您處理活動一樣。
    6. 按一下「儲存播放清單中的變更」按鈕。
    7. 按一下畫面右上角的工具 (扳手) 按鈕。然後清除快取。

    此時，您的活動會在 Brightcove Beacon 應用程式中建立為容器，如左圖所示 (影像上的事件名稱和圖片僅供舉例用途)，按一下您的活動，您就會看到類似右側影像的內容。要真正開始串流直播視頻，您需要在視頻雲端直播平台中設置您的傳輸。

    在工作室創建一個直播活動

    請依照下列步驟建立實際的直播活動，將會串流至 Brightve Beacon。

    1. 登錄到Video Cloud Studio，然後單擊生活在導航標題中。
    2. 從「管理事件」頁面中，按一下「建立事件」。的建立現場活動頁面將會打開。
    3. 輸入活動詳細資料。
      • 活動名稱 -活動名稱；如果直播事件另存為視頻資產，則此名稱也將出現在媒體模塊中（最多255個字符）
      • 簡短說明 -事件的摘要 (最多 250 個字元)
      • 標籤 -與事件關聯的標籤；如果將現場直播事件另存為視頻資產，則會將標籤分配給視頻（最多128個字符和1200個標籤）
      • 參考 ID -事件的參考 ID；必須是唯一的 (最多 150 個字元)
    4. 請點擊允許的如果您希望將直播活動流式傳輸到社交媒體平台，請單擊不允許。可以做到一旦事件開始流式傳輸或使用社會模塊。
    5. 選取「即時內嵌設定檔」。下拉式清單會列出所有即時內嵌設定檔。內嵌描述檔將控制所建立的即時轉譯的品質和數目。按一下「管理設定檔」連結將會在其他瀏覽器標籤中開啟內嵌設定檔編輯器。請注意，您必須是帳戶管理員才能存取內嵌設定檔編輯器。按一下轉譯連結，檢視將使用選取的內嵌輪廓建立的轉譯。
      三個層級的位元率和帳單詳細資料如下：
      • 第 1 級允許高達 5,500 Kbps，並以 1.0 小時計費
      • 第 2 層允許高達 10,000 Kbps，並以 1.5 小時計費
      • 第 3 級允許高達 20,000 Kbps，並以 2.0 小時的方式進行充電
    6. 選擇完成後將事件轉換為視頻資產將直播活動的VOD版本保存到您的Video Cloud帳戶。選取此選項後，會使用所選「即時內嵌設定檔」中的最高品質轉譯作為 VOD 視訊的來源。建立轉譯時，會使用帳戶的預設內嵌設定檔。請注意，即時活動結束後就無法使用 VOD 版本，視訊會在擷取程序完成後才能使用。
      注意：使用週期性事件時，不支援此選項。
    7. 請點擊高級選項並查看高級選項。只有其中一個需要你的注意。
    8. 點擊啟用流狀態通知複選框並添加Brightcove Beacon CMS的通知URL（您的Brightcove支持代表將提供此URL）。
    9. 按一下「建立事件」以啟動即時事件。[控制室] 頁面將會開啟，其中包含即時事件的詳細資料，包括編碼器將使用的串流端點 (RTMP URL) 和串流名稱。處理串流需要大約一分鐘的時間。
      若要在直播平台中開始串流，您需要編碼器，如需有關如何使用編碼器的資訊，請閱讀以下文件：

      逐步：使用即時模組與電流線上傳送

      逐步：使用開放式廣播軟體 (OBS) 的直播模組

    10. 在裡面控制室，複製工作編號和影片編號來自測試事件面板中，您都需要在Brightcove Beacon平台設置中配置實時事件。
    11. 接下來，返回Brightcove Beacon，然後單擊事件標籤。選擇您之前在此處創建的事件，然後在基本數據設置粘貼工作編號您的直播活動。
    12. 轉到流標籤，然後通過點擊+添加新流按鈕。
    13. 在裡面流視頻ID字段，粘貼影片編號從您先前創建的視頻雲事件中獲取。同時填寫以下有關該流的信息：
      • 選擇您可以在其中發佈視訊的裝置數目。
      • 設置流類型至生活從下拉菜單中。
      • 然後點擊更新事件按鈕，然後再離開頁面。
    14. 按一下畫面右上角的工具 (扳手) 按鈕。然後清除快取。
    15. 打開您的 Brightcove Beacon 應用程序並選擇您的活動（圖片上的活動名稱和圖片僅供例如用途）。
    16. 點擊玩按鈕，您現在應該可以在Brightcove Beacon App中看到來自視頻雲的實時事件流。

    活動結束後，您可以將所有或部分內容儲存為影片雲端中的影片資產，或是可在 Brightcove Beacon 中顯示的可播放串流。此程序將在本文件的下一節中詳細說明。

    從直播活動建立剪輯片段

    如果您想保留視頻以便以 VOD 錄製的視頻進行流媒體，您必須創建您的 Brightcove Beacon 直播活動的剪輯。

    Live 模組提供從已完成的事件建立視訊剪輯的功能。您也可以在現場活動進行時建立剪輯片段。

    如需完整的剪裁功能清單，請參閱使用即時模組建立視訊片段的說明文件。

    1. 在實時模塊中，轉到創建剪輯標籤在左側面板中。
    2. 使用剪裁按鈕或快速鍵，設定剪輯片段的起點和終點。您也可以點擊保存按鈕以保留整個視頻。
    3. 輸入剪輯視訊屬性。
      • 剪輯片段名稱 -剪輯片段的名稱
      • 簡短描述 -剪輯片段的說明
      • 標籤 -要與剪輯片段建立關聯的標籤
      • 參考 ID -指定給剪輯片段的參考 ID
      • 刪除源廣告中斷內容 -此選項可刪除剪輯中觸發的廣告中斷，僅保留節目內容
        注意：只有在串流啟用 SSAI 時，才會出現「移除來源廣告片段」核取方塊。
    4. 按一下「建立剪輯」。剪輯片段的轉碼程序將會開始。剪輯片段會使用帳戶的預設內嵌設定檔進行轉碼。

    將您的剪輯片段添加到布萊特灣信標作為可播放的串流影片

    在這裡，您將使用新創建的剪輯視頻的 ID，並使其成為可以在布萊特灣信標播放的流。

    1. 仍在Video Cloud Studio的實時模塊中時，複製號碼ID您創建的剪輯。
    2. 在Brightcove Beacon平台中，轉到大事記標籤，然後選擇流菜單。你會看到以下形式：
    3. 粘貼ID您剛剛從Studio的Live平台複製到了流視頻ID領域。
    4. 來自流類型下拉式變更生活至流。
    5. 按一下「基本資料」標籤，並將狀態從「已結束」變更為「已完成」。
    6. 離開之前，請點擊更新事件按鈕位於頁面底部。這將使玩 Brightcove Beacon應用程序中的可用按鈕可查看事件的記錄。

    (靜態進入點) SEP 即時事件

    舉例來說，「SEP Live 事件」是一種週期性事件，您可以在活動重新開始時啟動/停用，例如每週節目。此事件可以用作所有事件現場工作ID可以在Brightcove Beacon應用程序中傳輸到多個頻道。

    • 請注意，當您建立週期性事件時，會在「媒體」模組中自動為其建立視訊。請勿刪除此視訊，因為每次事件重複發生時都需要這個影片。
    • 在編碼器啟動之前，應該啟動週期性事件。

    請依照下列步驟在布賴特灣信標中建立 SEP (靜態入口點) 即時事件。

    1. 在實時平台中創建一個新活動。有關如何創建活動的更多信息，請閱讀創建一個新事件的部分使用實時模塊創建和管理實時事件文件。
    2. 展開高級選項，然後啟用使用靜態入口點（SEP）將其創建為重複事件選項。有關SEP的更多信息，請參見處理重複事件的部分使用實時模塊創建和管理實時事件文件。
    3. 如果您已經有SEP事件，請點擊支持標籤，您將能夠激活/停用它來自實時控制室。
    4. 複製您的現場工作ID。
    5. 返回Brightcove Beacon平台並通過粘貼工作編號重複發生的事件。還提供一個名稱和活動開始日期。
    6. 活動的日期和時間必須採用以下格式。
    7. 按一下「更新事件」按鈕。
    Brightcove Beacon CMS正在自動執行狀態實況事件的轉換，但在靜態入口點中，您可能有多個Brightcove Beacon事件與同一SEP相關聯。如果自動化出現問題，因為自動化運行較晚且重疊，則可以通過手動更改狀態。例如，來自可用的至已完成在裡面狀態領域。

    您可以在 Brightcove Beacon 平台中為您想要的所有即時活動執行此程序。所有這些傳輸都會從相同的即時傳輸串流，但每個傳輸的日期和時間都會有不同的事件設定中調整的日期和時間。
